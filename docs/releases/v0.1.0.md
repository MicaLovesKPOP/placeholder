# InputFlow v0.1.0

First technical preview release for InputFlow.

InputFlow is a Windows tray app for precise multilingual input-profile workflows. This preview is intentionally narrow: it validates the core switching engine, Korean Hangul/native mode safety, diagnostics, and portable Windows packaging before the settings UI and installer work begin.

## Validated Scope

- Toggle between Dutch (Netherlands) / United States-International and Korean Microsoft IME.
- Enter Korean Hangul/native mode using state-setting IME APIs instead of a blind Hangul-key toggle.
- Use `RightAlt` as a single-key trigger when configured.
- Return from Korean to the configured fallback profile.
- Keep config and logs in stable per-user paths:
  - `%APPDATA%\InputFlow\inputflow.json`
  - `%LOCALAPPDATA%\InputFlow\inputflow.log`
- Copy diagnostics from the tray menu, including installed input profiles and configured profile match reports.
- Run as a single tray instance per Windows session.

## Manual Windows Validation

Validated before release:

- Dutch (Netherlands) / US-International -> Korean Microsoft IME + Hangul/native mode.
- Korean Microsoft IME -> Dutch (Netherlands) / US-International.
- Korean Latin/A mode -> Korean Hangul/native mode after trigger.
- Tray `Copy Diagnostics`.
- Per-user config/log paths.
- Duplicate launch exits without creating a second tray instance or second hotkey handler.

## Requirements

- Windows 10 or Windows 11.
- .NET Desktop Runtime 8 for the framework-dependent portable build.

## Notes For Testers

The generated default config is seeded for the validated Dutch/Korean workflow. If your Windows input profiles differ, start InputFlow, use tray `Copy Diagnostics`, and update `%APPDATA%\InputFlow\inputflow.json` with the profile identifiers reported by your machine.

The default `RightAlt` trigger suppresses normal Right Alt / AltGr behavior while InputFlow is running. Change the trigger if you need AltGr for typing characters.

## Known Limitations

- No settings window yet.
- No first-run setup yet.
- No installer or auto-update flow yet.
- Manual JSON editing is required for non-seed workflows.
- Hold-to-switch and previous-profile workflow modes are not implemented yet.
- Korean Hangul/native mode is the only intentionally supported IME mode adapter so far.
- Some Windows language/input setups may require more exact TSF profile matching.
- Elevated apps may not accept input-method changes from a non-elevated InputFlow process.
- The project does not have a selected license yet.
