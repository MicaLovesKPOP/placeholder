# InputFlow v0.2.0

First multilingual preview release for InputFlow.

This release moves InputFlow beyond the original Dutch/Korean seed setup. It adds first-run setup, installed-profile picking, workflow editing, profile health diagnostics, setup recovery, and process exclusions while preserving the lightweight tray-app model.

## Highlights

- Fresh installs create an inventory-backed starter config and open `Setup Status`.
- `Setup Status` can add, edit, and remove configured profiles.
- `Setup Status` can add, edit, and remove workflows for:
  - toggle
  - direct switch
  - cycle
  - previous profile
- `Setup Status` can add and remove excluded processes.
- Single-key trigger setup warns that the key will be suppressed while InputFlow runs.
- Profile matching reports `matched`, `missing`, `ambiguous`, and `changed` states.
- Ambiguous profiles are not used for runtime switching.
- `Reset Setup` backs up the current config and writes a fresh starter config.
- Tray menu can enable or disable Start with Windows for the current user.
- Setup dialogs support basic keyboard operation and accessibility labels.

## Validation Scope

Before tagging this release, complete the release checklist in `VALIDATION.md`, including:

- core tests
- Release app build
- Windows x64 publish
- first-run setup smoke test
- toggle, direct-switch, cycle, and previous-profile workflow smoke tests
- Korean Hangul/native smoke test when Korean Microsoft IME is installed
- Copy Diagnostics inspection
- Reset Setup backup/recovery check
- single-instance check

## Requirements

- Windows 10 or Windows 11.
- .NET Desktop Runtime 8 for the framework-dependent portable build.

## Known Limitations

- This is still a technical preview for multilingual power users, not a daily-driver stable release.
- There is no installer or auto-update flow yet.
- There is no full standalone settings app yet; setup lives in the tray `Setup Status` window.
- Hold-to-switch workflow mode is not implemented yet.
- Profile groups and per-app remembered profile behavior are not implemented yet.
- Korean Hangul/native mode is the only intentionally supported IME mode adapter so far.
- Browser address bars and other application-owned chrome fields may not refresh IME composition state even when Windows reports the requested profile and mode are active.
- Elevated apps may not accept input-method changes from a non-elevated InputFlow process.
- Some Windows language/input setups may require more exact TSF profile matching in a future release.

## Upgrade Notes

- Existing v1 hotkey configs are migrated in memory to v2 workflows.
- Existing per-user configs remain in `%APPDATA%\InputFlow\inputflow.json`.
- Legacy `inputflow.json` files next to `InputFlow.exe` are copied into the per-user config path only when no per-user config exists.
- Manual JSON remains supported.
